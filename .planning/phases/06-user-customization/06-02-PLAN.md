      - `selectedRouteIndex?: number | null` — which route is selected (-1 or null = none)
      - `onRouteSelect?: (index: number | null) => void` — callback when route clicked
      - `onRouteMove?: (index: number, newPoints: PathPoint[]) => void` — callback when route dragged
    - Determine if route-interactive: `const isRouteInteractive = !!(onRouteSelect && onRouteMove)`

    **ArenaCanvas.tsx — Hit testing for route selection (per D-06):**
    - Add helper `findNearestRoute(point: PathPoint, paths: ArenaPath[]): number | null` that:
      - For each path, computes minimum distance from point to any line segment in the path
      - Returns the index of the nearest path if distance < threshold (e.g., 0.04 in normalized coords)
      - Returns null if no path is close enough
    - On pointerDown (when isRouteInteractive):
      - Convert pointer to normalized coords via fromCanvasCoords
      - Call findNearestRoute to check if a route was clicked
      - If hit: call `onRouteSelect(hitIndex)`, store the pointer start position in a ref for drag tracking
      - If miss: call `onRouteSelect(null)` to deselect

    **ArenaCanvas.tsx — Drag to move route (per D-08, D-09):**
    - Add refs: `dragStartRef = useRef<PathPoint | null>(null)`, `isDraggingRoute = useRef(false)`
    - On pointerDown (after selection): set dragStartRef to current normalized point, isDraggingRoute = false
    - On pointerMove (when selectedRouteIndex is set and dragStartRef exists):
      - Compute delta = current - dragStart
      - If delta > small threshold (0.01): set isDraggingRoute = true
      - Translate all points of the selected path by delta
      - Per D-09: Clamp each translated point to 0-1 bounds
      - Re-draw canvas with translated points (update a local drag offset state or ref)
    - On pointerUp:
      - If isDraggingRoute: call `onRouteMove(selectedRouteIndex, translatedPoints)` with final positions
      - Reset dragStartRef, isDraggingRoute

    **ArenaCanvas.tsx — Visual feedback for selection (per D-06, D-07):**
    - In the draw function, when rendering paths:
      - If `i === selectedRouteIndex`: draw with lineWidth 4.5 (instead of 2.5), add a subtle glow effect (draw same path again with lineWidth 6 at 30% opacity)
      - Selected route's label always visible and slightly larger font (11px instead of 9px)
    - Change cursor to "pointer" when hovering over a route (can approximate by checking on pointerMove and setting canvas cursor style)

    **ArenaRouteView.tsx — State management for route interaction:**
    - Add `const [selectedRouteIndex, setSelectedRouteIndex] = useState<number | null>(null)`
    - Add handler `handleRouteMove = useCallback((index: number, newPoints: PathPoint[]) => { ... })`:
      - Create a copy of arenaPaths, replace the moved route's points with newPoints
      - Call `onRoutesGenerated(updatedPaths)` to persist to wizard-store (uses existing prop)
      - This triggers re-render since arenaPaths in store changes
    - Pass new props to ArenaCanvas: `selectedRouteIndex={selectedRouteIndex}`, `onRouteSelect={setSelectedRouteIndex}`, `onRouteMove={handleRouteMove}`
    - When `selectedRouteIndex` is set, show the selected exercise name prominently above the canvas (small info bar)
    - Clear selection when programOrder changes (useEffect that sets selectedRouteIndex to null)

    **Important:** The route-interactive mode is SEPARATE from the existing drawing-interactive mode. Do not break the existing onMouseDown/onMouseMove/onMouseUp drawing props (used by ArenaEditor if it still exists).
