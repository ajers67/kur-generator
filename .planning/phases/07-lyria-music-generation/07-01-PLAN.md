    3. Create `src/lib/lyria-provider.ts` (per D-02):
       ```typescript
       import type { MusicProvider } from "./music-provider";

       export class LyriaProvider implements MusicProvider {
         async generateTrack(prompt: string, bpm: number, durationSec: number): Promise<ArrayBuffer> {
           const res = await fetch("/api/music/generate", {
             method: "POST",
             headers: { "Content-Type": "application/json" },
             body: JSON.stringify({ prompt, bpm, durationSec }),
           });
           if (!res.ok) {
             const text = await res.text().catch(() => "");
             throw new Error(`Music generation failed (${res.status}): ${text}`);
           }
           return res.arrayBuffer();
         }
       }
       ```
       This is a thin client — all Lyria WebSocket logic is server-side in the API route.

    4. Create `src/lib/pcm-to-wav.ts` — standalone WAV encoder for raw Int16Array PCM data:
       - `concatInt16Arrays(arrays: Int16Array[]): Int16Array` — concatenates multiple chunks
       - `pcmToWav(pcmData: Int16Array, sampleRate: number, channels: number): ArrayBuffer` — wraps PCM in WAV header (44-byte RIFF/WAV header, PCM format=1, 16-bit)
       - Lyria outputs 48kHz 16-bit stereo (2 channels). The WAV header must reflect this exactly.
       - Do NOT reuse `audioBufferToWav` from audio-mixer.ts — that takes AudioBuffer (browser API), this takes raw Int16Array (server-side compatible).

    5. Create `src/lib/__tests__/pcm-to-wav.test.ts`:
       - Test that pcmToWav produces correct RIFF header ("RIFF" at offset 0, "WAVE" at offset 8)
       - Test that fmt chunk has PCM format (1), correct channels, correct sample rate
       - Test that data chunk length matches input PCM length * 2 (bytes per sample)
       - Test concatInt16Arrays concatenates correctly

    6. Create `src/lib/__tests__/lyria-provider.test.ts`:
       - Mock global `fetch` to verify LyriaProvider sends correct POST to `/api/music/generate`
       - Verify request body contains {prompt, bpm, durationSec}
       - Verify error thrown on non-ok response
       - Verify ArrayBuffer returned on success
