
    ```typescript
    interface GaitTrack {
      gait: string;
      prompt: string;       // Editable prompt (auto-generated default, user can edit per D-13)
      bpm: number;          // Auto-set from GAIT_BPM_TARGETS per D-07
      audioBlob: Blob | null;
      audioUrl: string | null;  // URL.createObjectURL for <audio>
      generating: boolean;   // Loading state per D-20
      error: string | null;  // Error state per D-22

    const [genre, setGenre] = useState("Klassisk");  // Global genre per D-10
    const [tracks, setTracks] = useState<GaitTrack[]>([]);  // One per gait per D-09
    const [generatingAll, setGeneratingAll] = useState(false);  // Per D-23
    ```

    **Initialize tracks** from unique gaits in programOrder (same logic as current: filter out "overgang"). Each track gets auto-generated prompt via buildDefaultPrompt. Use useEffect to reinitialize when genre changes (update prompts only if user hasn't edited them — track with `promptEdited` flag, or simpler: just update on genre change since it's a new generation context).

    **Layout (per D-25):**

    1. Header: "Musik" + intro text ("Generer musik til hver gangart med AI. Vaelg genre og tilpas prompts.")

    2. Genre selector at top (per D-10): styled dropdown/select with MUSIC_GENRES options

    3. "Generer al musik" button (per D-23): generates all gaits sequentially. Disabled during generation.

    4. One card per gait (per D-25), each containing:
       a. Gait name + colored dot (using GAIT_COLORS, GAIT_LABELS)
       b. BPM badge showing target BPM (e.g., "57 BPM")
       c. Textarea/input for prompt (per D-11, D-13) — pre-filled with buildDefaultPrompt, editable
       d. "Generer" button — calls generateTrack for this gait. Disabled during generation.
       e. Loading state (per D-20, D-21): animated pulse/spinner + "Genererer musik... (~10-15 sek)"
       f. Error state (per D-22): red text + "Prov igen" button
       g. Audio player when track exists (per D-14, D-15, D-16):
          - `<audio>` element with src={audioUrl}
          - Play/pause button
          - Progress bar: styled div that updates via timeupdate event on audio element (per D-15, discretion: simple progress bar, not wavesurfer)
          - Volume slider (per D-16): `<input type="range" min="0" max="1" step="0.1">` controlling audio.volume
          - Duration display
       h. "Generer igen" button when track exists (per D-17): shows confirmation dialog before replacing (per D-19, use window.confirm "Erstat nuvaerende musik?")

    5. Disclaimer at bottom: "Musik genereret med Google Lyria AI. Instrumentalmusik beregnet til dressur freestyle."

    6. "Tilbage" button calling onBack (same as current)

    **Generation logic:**

    Per-gait generation (`handleGenerate(gaitIndex)`):
    1. Set track.generating = true, track.error = null
    2. Create provider: `const provider = createMusicProvider()`
    3. Call `provider.generateTrack(track.prompt, track.bpm, TRACK_DURATION_SEC)`
    4. On success: create Blob from ArrayBuffer (`new Blob([arrayBuffer], { type: "audio/wav" })`), create URL via `URL.createObjectURL`, update track state (per D-08, D-18)
    5. On error: set track.error with message, track.generating = false (per D-22)
    6. Clean up old audioUrl via `URL.revokeObjectURL` before setting new one (prevent memory leak)

    Generate-all (`handleGenerateAll`) per D-23:
    1. Set generatingAll = true
    2. Loop through tracks sequentially (not parallel — avoid rate limits)
    3. Call handleGenerate for each gait
    4. Set generatingAll = false when done

    Re-generation (`handleRegenerate(gaitIndex)`) per D-17, D-18, D-19:
    1. If track has audioBlob, show `window.confirm("Erstat nuvaerende musik?")`
    2. If confirmed, call handleGenerate(gaitIndex) — replaces previous track

    **Audio playback state:**
    Use one useRef per component for the currently playing audio element. Track playing state per gait. On play, pause any other playing track first. Use timeupdate event to update progress bar position. Use loadedmetadata to get duration.

    **Cleanup:** In useEffect cleanup, revoke all object URLs to prevent memory leaks.

    **Follow project conventions:**
    - "use client" at top
    - Named export: `export function MusicManager({ level, programOrder, onBack }: Props)`
    - Props interface named `Props`
    - useCallback for handlers passed as props or used in effects
    - Tailwind classes for all styling
    - Danish UI text throughout
    - Error handling with try/catch per existing patterns
