    Per D-01: export function validateProgram(level: KurLevel, programOrder: Exercise[]): ValidationResult[]

    Per D-02: file is src/lib/rule-validator.ts

    Per D-03: export interface ValidationResult {
      rule: string;        // e.g. "missing-exercise", "forbidden-exercise", "min-distance", "entry-position"
      severity: "error" | "warning";
      message: string;     // Danish, specific — per D-13
      exerciseIds?: number[];
    }

    Per D-04: "error" for hard violations (RULE-01 missing, RULE-03 forbidden, D-08 order), "warning" for advisory (RULE-02 distance)

    Implementation rules per CONTEXT decisions:
    - D-05 (RULE-01): Compare level.exercises IDs against programOrder IDs. Any level exercise not in program = error. Message: "{exercise.name} mangler i programmet"
    - D-06 (RULE-02): For each exercise in programOrder that has minDistance, emit warning. Message: "{exercise.name} skal daekke {minDistance}" — this is advisory since actual distance cannot be measured without arena routes
    - D-07 (RULE-03): For each exercise in programOrder, check if its id exists in level.exercises. If not = error. Message: "{exercise.name} er ikke tilladt pa dette niveau"
    - D-08: Check programOrder[0] is the entry exercise (name includes "Indridning"). If not, error: "Indridning skal vaere forste ovelse"
    - D-08 additional: Check programOrder[last] is the entry/finale. For levels where the entry exercise name also contains "afslutningsparade", verify it appears at position 0. For the last-position check: the last exercise should be the one containing "Indridning" (since in all current levels, the entry exercise also serves as the finale marker). If the entry exercise is at position 0 but the level only has one overgang exercise, skip the last-position check (entry IS the finale, and it must be first per D-08).
