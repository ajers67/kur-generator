  <action>
    Rewrite `src/lib/program-generator.ts` completely. Keep the same named export `generateProgramOrder`.

    **New function signature (per D-21, D-22, D-23, D-24):**
    ```typescript
    interface GeneratorOptions {
      seed?: number;
    }

    export function generateProgramOrder(
      level: KurLevel,
      ratings: Record<number, StrengthRating>,
      temperament: "calm" | "neutral" | "energetic",
      options?: GeneratorOptions
    ): Exercise[]
    ```

    **Internal structure — implement these as module-private helper functions:**

    1. **`mulberry32(seed: number): () => number`** (per D-20)
       - Returns a function that produces pseudo-random floats 0-1
       - If no seed provided, use `Math.random` as the RNG

    2. **`fisherYatesShuffle<T>(arr: T[], rng: () => number): T[]`** (per D-17)
       - Returns a new shuffled array (does not mutate input)
       - Uses the provided RNG function

    3. **`detectPairs(exercises: Exercise[]): [Exercise, Exercise][]`** (per D-14)
       - Finds left/right pairs by matching exercise names
       - Strategy: normalize name by removing "til venstre"/"til hojre", "til hoejre", "(H->V)"/"(V->H)", "(H og/eller V)" variants
       - Two exercises are a pair if their normalized names match AND they have different original names
       - Return array of tuples

    4. **`scoreExercise(exercise: Exercise, rating: StrengthRating, coefficient: number): number`** (Claude's discretion on exact formula)
       - Score formula: `(ratingScore * 2) + (coefficient * 3)`
       - Where ratingScore: strength=3, neutral=1, weakness=0
       - Higher score = more prominent placement (climax)
       - Exception per D-09: if coefficient >= 2 AND rating === "weakness", cap score at midrange (score = 4, placing in build-up not climax)

    5. **`assignArcSections(exercises: Exercise[], ratings, temperament, rng)`** — the core algorithm:

       a. **Extract fixed exercises:**
          - Find entry exercise: name includes "Indridning" (per D-02). Remove from pool.
          - Find finale: For levels where entry is also the finale (e.g., LA where id=14 is both), the entry exercise serves double duty — it goes first AND the program simply ends after all other exercises. For levels with a separate halt/hilsen, extract it. Remove from pool.

       b. **Group remaining exercises by gait** (skridt, trav, galop, passage, piaffe, overgang)

       c. **Order gait groups by temperament (per D-10, D-11, D-12):**
          - calm: ["skridt", "trav", "galop"]
          - energetic: ["trav", "galop", "skridt", "galop"] (split galop: first half early, second half after skridt pause)
          - neutral: ["trav", "skridt", "galop"]
          - Passage/piaffe always assigned to climax section (per D-13)
          - Remaining "overgang" exercises distributed among sections

       d. **Build ordered pool** following gait sequence, then within each gait group sort by score (computed in step 4)

       e. **Divide into arc sections (per D-01):**
          - Total exercises in pool = N
          - Build-up: first ~35% of N (rounded)
          - Climax: next ~30% of N (rounded) — the scored-highest exercises go here (per D-04, D-06, D-08)
          - Wind-down: remaining ~35% of N — lower-scored exercises (per D-05, D-07)
          - Passage/piaffe forced into climax (per D-13)

       f. **Place exercises by score within each section:**
          - Sort each section's exercises by score descending
          - Climax gets the highest-scored exercises
          - Build-up gets moderate scores
          - Wind-down gets lowest scores (weakness exercises land here per D-07)

       g. **Shuffle within sections** using Fisher-Yates with RNG (per D-17, D-18)

       h. **Handle symmetry pairs (per D-14, D-15, D-16):**
          - After shuffle, for each detected pair, if they ended up more than 3 positions apart, move the second one closer (within 2-3 positions)
          - If they ended up adjacent, insert one exercise between them
          - If one side is strength-rated, ensure it comes first in the pair (per D-16)

       i. **Assemble final program:**
          - [entry] + [build-up] + [climax] + [wind-down] + [finale if separate from entry]
          - Deduplicate (filter seen IDs)

    **Important edge cases:**
    - LA level: exercise 14 is BOTH entry and finale. Place it first. No separate finale.
    - Levels with very few exercises (LA has 14): sections may have 3-5 exercises each — that's fine.
    - If no exercises have passage/piaffe gait, climax section just uses highest-scored exercises from other gaits.
    - If no seed provided, use `Math.random` so each call varies (per D-04 anti-pattern avoidance).

    The function MUST remain pure (per D-24): no side effects, no store access, no DOM access.
